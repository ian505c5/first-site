Project Overview

One of our first major projects in Web Development 2: Responsive Design was to recreate the Touch Agency's website based on a set of photoshop comps. Each comp represented a different layout based on what resolution the user was viewing the site at. Thus in order to recreate this effect, we used CSS media queries to target specific breakpoints where the layout would adjust.

Introduction

Touch's agency site is a great example of smooth responsive design, and features breakpoints at multiple resolutions. This allows the users to have a consistent viewing experience across a variety devices and orientations.

The Process

The developer at Touch had a little more experience than I did at the time of this writing, and relied on Javascript to arrange and sort their elements. We were limited to only HTML/CSS, and thus had to be creative in our solution. The way I went about was to create .div classes for the largest common boxes. The next step was simply a matter of organizing the the smaller elements within these boxes. And finally, you set all the large container to float: left, and if you followed your comp specs correctly, the boxes should fall into place at each break point. Also, I really liked Touchâ€™s rollover animations, but again, they used Javascript which is a little over my head at this point. So I attempted to mimic them with a :hover box-shadow and opacity style.

Final Thoughts

This was a definite challenge compared to previous assignments. It really showed that while responsive design may be a pain to layout at first, if you approach it methodically with a plan in hand, the pieces tend to fall into place relatively easily.
